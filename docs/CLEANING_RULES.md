# Clipboard Cleaner — 清洗规则速查

> 一页纸看完所有清洗映射。改规则前先在这里登记。
> 详细实现见 [`cleaner.py`](../cleaner.py)，详细架构见 [`TECHNICAL_DESIGN.md`](TECHNICAL_DESIGN.md)。

清洗目标：把 Claude Code / Ghostty 终端复制出来的内容，转成可以直接粘到**微信、飞书、Slack** 等不渲染 Markdown 的 IM 工具里的干净文本。

---

## 一、行内标记映射（IM 模式核心）

| Markdown 原文 | 清洗后 | 原因 |
|---|---|---|
| `` `code` `` | `「code」` | IM 不渲染反引号；中文角引号视觉等价 |
| `**bold**` | `【bold】` | 强调通过中文方括号传达 |
| `__bold__` | `【bold】` | 同上，下划线变体 |
| `*italic*` | `italic`（去标记） | IM 中无法表达斜体，去标记保留文字 |
| `_italic_` | `italic`（去标记） | 同上 |
| `[文字](url)` | `文字 (url)` | 链接展开为可读形式 |
| `# H1` ~ `###### H6` | `【标题】` | 各级标题统一转角括号 |
| `## 标题 ##`（ATX 闭合） | `【标题】` | 尾部 `#` 同样剥除 |

**保护规则**：反引号 `` ` `` 内的内容会被先用占位符替换，等其他变换做完再还原——所以 `` `*foo*` `` → `「*foo*」`，反引号里的 `*` 不会被斜体规则误伤。

**代码块内**：`*`、`**`、`` ` ``、`[](url)` **全部不变换**，字面保留。

## 二、块级处理映射

| 类型 | 识别方式 | 处理 |
|---|---|---|
| 代码围栏 ` ``` ` / `~~~` | 行首匹配 `_FENCE` | 围栏行**整行去除**，内部代码原样保留 |
| 标题 `#`~`######` | 行首匹配 `_HEADING` | 转 `【标题】`（见上表） |
| 无序列表 `- * •` | 行首匹配 `_UNORDERED_LIST` | 保留 marker，对内容做行内变换 |
| 有序列表 `1. 1) 1、` | 行首匹配 `_ORDERED_LIST` | 同上 |
| 列表延续行（缩进续行） | 同 list 组内的纯缩进行 | 合并到上一项末尾 |
| Markdown 表格 `\| a \| b \|` | 行首 `\|` 且包含分隔符 | 转**数字条目列表**：`1. 列1：值1\n   列2：值2` |
| Box-drawing 表格 `┌─┬─┐` `\| ├─┼─┤` `\| └─┴─┘` | `_is_box_table_line` 高优先级识别 | 转数字条目列表（同 Markdown 表格）；单行装饰性 box 退化为去边框留内容 |
| 引用单层 `> text` | 行首 `>` `▎` `\|` `｜` `│` | 装饰剥除，连续行按硬换行规则合并 |
| 引用嵌套 `> > text` | 多个装饰字符 | 装饰剥除，**整组先合并再包一对** `『 』`（嵌套层数决定包几层） |
| ASCII / box drawing 边框整行 | `_IS_BORDER` 全行匹配 | **整行删除**（box-drawing 表格走更高优先级） |
| 水平分割线 `---` `***` | 落入边框规则 | 删除（IM 中没意义，靠空行已经能分段） |
| Obsidian callout `[!tip] 标签` | `_process_callout` | 转 `## 标签` → 最终渲染为 `【标签】` 独立成行；纯 `[!info]` 无标签则整行消除 |
| YAML front-matter `---\nkey: val\n---` | `_strip_frontmatter` 在 `clean()` 入口剥除 | 标准格式整段消除；GPT 误生成的伪格式（`---  title: ...` 或紧贴的 `---title: ...`）也兜底消除直到首个空行 |
| Claude Code 输出前缀 `⏺` `⎿` | `_strip_cc_prefix` 在 `clean()` 入口剥除 | 终端输出气泡 bullet，复制时被一起拷走；进 IM 没意义，整体去掉 |
| 普通段落 | 其他 | 去公共缩进 + 保守合并硬换行 + 行内变换 |
| 空行 | 全空白 | 连续多空行压成 1 空行（首尾空行去掉） |

## 三、硬换行合并规则（保留优先）

默认**合并**两行，只在以下情况**保留**换行：

1. 上下行至少一个是空行
2. 上一行以**强终止标点**结尾：`。！？.!?:：`
3. 下一行是 Markdown 结构开始：` ``` ` / `## 标题` / `- 列表` / `\| 表格` / `---`
4. 下一行以大写字母开头且首词不是常见小词（the/a/is/are/this/it/...）
5. 下一行以 `数字+点` 开头（疑似有序列表）

**合并优先例外**：上一行以 **CJK 字符** 结尾且无强终止标点 → **强制合并**，无视下一行大小写。
这条规则解决"详见这个\nGitHub Issue"这种中文段落里夹英文专有名词的硬换行被规则 #4 误保留的问题。

合并时的空格策略（CJK 友好）：
- 上一行末尾是 **CJK 字符** + 下一行开头是 **CJK / 中文标点** → 直接拼接，不加空格
- 其他情况 → 合并时插入一个空格（英文需要词间空格）

## 四、特殊空白处理

| 处理 | 时机 |
|---|---|
| 标准化换行符 `\r\n` `\r` → `\n` | 第一步 |
| 所有行 `rstrip()` | 第二步 |
| 公共缩进剥离 | 按块组内分别计算 |
| 续行缩进剥离（首行无缩进、续行 ≥2 空格） | normal 块内 |
| 多空行压缩 `\n{3,}` → `\n\n` | 末尾 |
| 首尾 `strip()` | 末尾 |

## 五、幽灵捕获过滤（has_format_artifacts）

**只有满足以下任一条件的剪贴板内容才会进面板**（避免语音输入、其他应用复制的干净文本误捕获）：

1. 所有非空行有 ≥ 2 空格公共缩进
2. 首行无缩进 + 至少 1 行续行有 ≥ 2 空格缩进（Claude/Ghostty 硬换行典型样式）
3. 行首存在引用装饰（`> ` `▎ ` `\| ` `｜ `）
4. 存在 ASCII / box drawing 边框整行
5. 存在 trailing spaces（行尾 ≥ 2 空格）或 tab
6. 存在代码块围栏（` ``` ` / `~~~`）

**取舍**：宁愿漏掉无格式痕迹的极短输出，也不要让"早上好"这种语音输入挤进面板。

## 六、加新规则的清单

每加一条新清洗规则，请：

1. 在 [`cleaner.py`](../cleaner.py) 实现
2. 在本文档对应的表格里登记
3. 在 [`tests/test_cleaner.py`](../tests/test_cleaner.py) 加单元测试（至少 1 个常见 + 1 个边界）
4. 如果是块级规则，考虑在 [`tests/fixtures/`](../tests/fixtures/) 加 golden fixture
5. 跑 `python3 -m pytest tests/ -v` 确认全绿
